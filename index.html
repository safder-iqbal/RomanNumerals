Roman Numeral Generator

Starting the mini-project

With a given range of between 1 and 3999, I decided the quickest way to implement this in the DRY-est possible way was to use recursion.

This meant using string concatenation and recursively calling the generate method. This meant that I only had to define the base clauses once and keep calling to slowly build up the string of numerals.

The only flaw with this approach meant this would not be easily extensible, and so it wouldn't scale well if the range were extended beyond 3999. This is because it would be liable to cause a stack overflow. Additionally the string concatenation is an expensive operation, since without using recursion a StringBuilder would be more efficient.

The alternate solution

The alternate solution, located in the branch named alternate_solution instead uses Enumerables and Tuples to iterate over a given roman numeral as many times as necessary, before then moving onto the next one.

To identify how many times a numeral needed to be iterated over, modulo over the various numeral values, and division to quickly generate a string that represents the number.

To try and keep to the DRY approach, that looks very WET at first glance, I moved the generation of the numerals into a seperate, helper method.

Assumptions & Caveats

As previously mentioned, the range is hard-coded to a number between 1 and 3999.
